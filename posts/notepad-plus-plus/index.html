Notepad Plus Plus

26 Dec 2025

Esta investigacion fue iniciada por una alerta generada en un EDR, “Possible Masquerading Behavior”.

Possible Masquerading Behavior: Un proceso archivo o conexion se esta haciendo pasar por otro:

  • Imita un nombre
  • Imita una ruta
  • Imita un comportamiento
  • Intenta ocultar su verdadera identidad.

El proceso:

C:\ProgramData\USOShared\svchost.exe -nostdlib -run C:\ProgramData\USOShared\conf.c

Este proceso fue lanzado desde el siguiente proceso padre:

“C:\Users\User\AppDatal\Roaming\Bluetooth\BluetoothService.exe” -k

Vamos a analizar el detalle de estos procesos:

De primera mano la ejecucion del proceso svchost desde ese origen nos dice con alto grado de certeza que estamos ante un malware, veremos que esta haciendo este comando, pero antes analizaremos el directorio USOShared:

-drwxrwxrwx 1 root root      0 Apr  1  2024 Logs
-rwxrwxrwx 1 root root   5249 Oct 23 06:30 conf.c
-rwxrwxrwx 1 root root 148992 Oct 23 06:31 libtcc.dll

No tenemos el ejecutable svchost.exe, pero gracias al EDR si podremos obtener informacion importante:

ImpHash: 64a1d9329f13d97b45a6c43b84ad78ca
md5: fd906f68ebb7cf181102e1ccc63a886a

sha256: 0f235d6df51c0f6c21d970a48fd528b8aa058ed110b1f4842391336dd324532d

Ningun de estos hashes tiene algun tipo de analisis en virustotal, esto nos deja con muy poca informacion, sin embargo hay un par de archivos en el directorio que si pueden ser muy interesantes:

Iniciemos por el directorio Logs:

drwxrwxrwx 1 root root 48K Oct 23 07:09 User

Al estar en C:\ProgramData\USOShared el malware se camufla dentro del proceso de actualizacion, USO quiere decir Update Session Orquestrator y es un componente de Windows Update, normalmente almacene logs de actualizacion dentro del mismo directorio, es por esto que existen los directorios User y System, con archivos de extension .etl, esta parte del analisis ya que el objetivo del malware al almacenarse dentro de este directorio es simplemente ocultarse dentro de procesos legitimos.

Estamos ante un shellcode embebido, una tecnica conocida de malware:

Que hace la siguiente funcion:

int _runmain()
{
    unsigned char temp[836] = { ... };
    ((int(*)())temp)();
}
  • Declara un array de bytes
  • Bytes en x86
  • Ejecuta el buffer cargado

Shellcode execution from RWX memory

  • x86 (32 bits)
  • Ejecutable windows
  • Position independent code (PIC)
  • API Resolving dinamico (No imports)
  • Network-enabled Payload

Resolucion dinamica de APIs

FC                cld
E8 89 00 00 00    call next
60                pushad
64 8B 52 30       mov edx, fs:[edx+0x30]   ; PEB
  • Accede al PEB (Process Environment Block)
  • Recorre InMemoryOrderModuleList
  • Encuentra kernel32.dll
  • Busca direcciones de funciones por hash

Hashing de funciones, el siguiente patron se observa varias veces:

AC
C1 CF 0C
01 C7

Esto es un hash rotativo tipico para:

  • LoadLibraryA
  • GetProcAddress
  • WinExec
  • VirtualAlloc
  • InternetOpenA
  • InternetConnectA
  • HttpOpenRequestA
  • InternetReadFile

El codigo no tiene imports visibles para evadir detecciones de analisis estatico, dentro del shellcode se pueden obtener strings en texto claro relacionados con el C2, aqui tenemos un nuevo IoC

api[.]wiresguard[.]com

Las cabeceras http completas son:

Host: api.wiresguard.com
Accept: */*
Accept-Language: en-US,en;q=0.5
Connection: close
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) ...

Esto implica que estamos ante un beacon que realiza el request http manual.

Ejecucion de PAYLOAD remoto:

68 00 10 00 00   ; MEM_COMMIT
68 00 00 40 00   ; PAGE_EXECUTE_READWRITE
FF D5            ; VirtualAlloc
  • Reserva memoria RWX
  • Copia bytes recibidos
  • Salta a la direccion

Asi mismo se identifica el string /users/admin lo cual esta asociado a la ruta http del C2, la alta entropia nos indica que el payload esta cifrado y se descifra en tiempo de ejecucion.

Dentro del mismo directorio se observa el archivo:

4a52570eeaf9d27722377865df312e295a7a23c3b6eb991944c2ecd707cc9906 libtcc.dll

Ningun reporte en virustotal, sin embargo esta libreria hace parte de un proyecto llamado tiny c compiler, comparando los hashes coincide perfectamente con la dll en el proyecto, la pueden encontrar en: https://download.savannah.gnu.org/releases/tinycc/

Esto confirma la compilacion de codigo en runtime, ahora que ya tenemos clara la ejecucion de malware y su commmand a control, vamos al parent process, ese parent process es BluetoothService.exe, el ejecutable ya no existe en el sistema, pero gracias a los registros de EDR, podremos identificarlo, en los logs SYSMON (System monitor)

Este ejecutable establecio contacto con la direccion IP 61[.]4[.]102[.]97, puerto 443, este beacon contacta 6 veces el C2 cada 60 segundos.

Teniendo en cuenta que la firma es valida, este archivo es usado por bitdefender para ejecutar la suscripcion y gestionarla:

  • Activar Bitdefender con una clave
  • Renovar la licencia cuando vence
  • Vincular el equipo a una cuenta de Bitdefender Central
  • Validar el estado de la suscripción con los servidores de Bitdefender
  • Sincronizar cambios de plan/licencia

Es importante tener en cuenta que durante el analisis tambien se identifico el archivo log.dll en la misma ruta de BluetoothService.exe, esto suena a un ataque ampliamente conocido como dll side loading, que consiste en colocar una dll maliciosa en el mismo directorio del ejecutable legitimo, en este baso BitDefender, en tiempo de ejecucion BluetoothService.exe primero carga la dll log.dll que se encuentra en el mismo directorio, logrando ejecutar codigo malicioso desde un binario legitimo.

Revisando los hashes veo que el sitio security.com, analizo un incidente con las mismas TTP y uno de los IoCs coinciden perfectamente con el ataque, precisamente coincide con nuestro BluetoothService.exe https://www.security.com/threat-intelligence/billbug-china-espionage este grupo logro asociar la APT con las TTP.

Tambien existe un analisis de Diciembre de 2025 en AnyRun con las mismas TTP https://any.run/report/0343a56b1e5c17ac0bbcded8e9b625704bb7a8ea31167ec702a40dd0dc03bb8d/c86e5379-1d0b-4112-9ede-146e49dd87fe
